# Roadmap

## Phase 0 — Canonical documentation

Goal: make the repository usable by humans and agents.

Tasks:

- Create product spec
- Create profile category schema
- Create question bank
- Create compatibility model notes
- Create safety/privacy notes
- Create MVP architecture and implementation plan
- Create branding notes

Exit criteria:

- A new agent can understand the project without reading the chat history.

## Phase 1 — Static prototype

Goal: demonstrate the experience without real matching infrastructure.

Tasks:

- Build landing page concept
- Build onboarding flow mock
- Build AI interview mock
- Build structured profile preview
- Build compatibility report mock
- Build sample match explanation

Exit criteria:

- Demo shows how this differs from swipe dating.

## Phase 2 — Functional profile builder

Goal: collect structured profile data from one user.

Tasks:

- User account model
- Profile category schema
- Question-answer storage
- AI follow-up generation
- Profile summary generation
- User review/edit flow

Exit criteria:

- A user can complete a deep AI-assisted profile.

## Phase 3 — Compatibility scoring prototype

Goal: compare two test profiles and generate a match explanation.

Tasks:

- Define scoring weights
- Implement hard constraint filter
- Implement dimension scoring
- Implement unknown/confidence scoring
- Generate explanation output
- Capture decline feedback

Exit criteria:

- System can compare two profiles and explain fit/risk.

## Phase 4 — Small closed test

Goal: test whether real users find the intake/report useful.

Tasks:

- Recruit small trusted test group
- Collect consent
- Run profile process
- Generate compatibility reports
- Gather feedback
- Refine categories/questions

Exit criteria:

- Users report that the system understands them better than normal dating profiles.

## Phase 5 — Safety, verification, and privacy

Goal: make the system safer before public launch.

Tasks:

- Identity verification plan
- Photo/video verification plan
- Relationship status verification options
- Consent and data deletion flow
- Sensitive field handling
- Abuse reporting
- Fraud detection basics

Exit criteria:

- System has credible safety/privacy foundation.

## Phase 6 — MVP launch candidate

Goal: first private beta.

Tasks:

- Real user accounts
- Secure database
- Matching queue
- Recommendation flow
- Feedback loop
- Admin moderation tools
- Basic analytics focused on outcomes, not engagement

Exit criteria:

- Private beta can run with real users under controlled conditions.

## Build principles

- Do not build swipe UI.
- Do not optimize for addictive engagement.
- Do not overpromise soulmate certainty.
- Do not hide uncertainty.
- Build evidence, explanation, and trust into every feature.
